## 问题分类与处理

### 工具类问题 (需要修复)
- 编译错误/警告
- NullPointerException
- AssertionError (非预期的数据库 bug)
- SQLSyntaxErrorException
- IgnoreMeException 未正确处理
- 类型转换错误
- Schema 读取失败

### 数据库类问题 (允许发现)
- 查询结果不一致 (这是 Oracle 的目标)
- 数据库逻辑 bug

## 修复策略

1. **编译问题**: 直接修复代码
2. **运行时异常**: 分析异常原因，添加预期错误或修复逻辑
3. **Schema 问题**: 检查 GaussDBPGSchema.java 的查询逻辑
4. **表达式问题**: 检查 GaussDBPGExpressionGenerator.java 的生成逻辑
5. **约束问题**: 检查 GaussDBPGTableGenerator.java 和错误处理

## 输出要求

每次迭代后输出：
1. 发现的问题列表
2. 已修复的问题列表
3. 当前状态评估

当所有 Oracle 测试通过（无工具类错误），输出：
```xml
<promise>ALL GAUSSDB-PG ISSUES FIXED</promise>
```

## 最大迭代次数

设置 max-iterations: 30
